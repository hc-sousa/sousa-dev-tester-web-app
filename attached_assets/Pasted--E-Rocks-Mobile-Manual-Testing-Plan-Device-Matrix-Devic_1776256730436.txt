# E-Rocks Mobile — Manual Testing Plan

## Device Matrix

| Device           | OS         | Screen Size | Notes                      |

Run every section in **dark mode** (default) and **light mode**.

## 1. Authentication & Account

### 1.1 Login (LoginScreen)
- [ ] Sign in with valid e-rocks credentials → lands on Settings or `returnTo` screen
- [ ] Sign in with **wrong password** → error alert, no crash
- [ ] Sign in with **empty fields** → validation prevents submit
- [ ] Tap show/hide password toggle → password toggles visibility
- [ ] Login triggers **MFA challenge** → navigates to MfaChallengeScreen
- [ ] Login returns `action_required` (legacy password) → navigates to LegacyPasswordSetup
- [ ] Login returns `upgrade_required` → navigates to HostedAuth with correct URL
- [ ] "Forgot Password" → opens external URL in browser
- [ ] "Register" → opens external e-rocks registration URL
- [ ] Privacy Policy / Terms links → open correct URLs
- [ ] Login from a **paywall prompt** → after success, returns to paywall / original screen
- [ ] Keyboard does not obscure inputs on small screens (KeyboardAvoidingView)

### 1.2 MFA Challenge (MfaChallengeScreen)
- [ ] Enter valid 6-digit code → completes auth, navigates to home/settings
- [ ] Enter wrong code → error message, stays on screen
- [ ] Leave code blank → submit blocked or clear error

### 1.3 Logout
- [ ] Logout from Settings → clears tokens, RevenueCat logout, tier resets to `free`
- [ ] After logout, subscription-gated content shows free-tier UI (blurred prices, lock icons)
- [ ] After logout, deep-linking back into a premium screen shows paywall

### 1.4 Session Persistence
- [ ] Kill app → reopen → user still logged in (tokens from expo-secure-store)
- [ ] Force-expire token → next API call triggers silent refresh (401 → refresh interceptor)
- [ ] If refresh fails → user is logged out gracefully

## 2. Subscription & Paywall

### 2.1 Tier Gating
- [ ] **Free user** sees blurred/locked prices on ItemBrowseCard, ItemDetailScreen, PriceDisplay
- [ ] **Free user** tapping a locked feature triggers paywall or login prompt
- [ ] **Premium (Collector) user** sees all prices, market data, associations, full localities
- [ ] Tier label in AccountHubHeaderButton shows "Free" or "Collector" with correct color

### 2.2 RevenueCat Integration
- [ ] Purchase flow (sandbox): subscribe → tier updates to `premium` immediately
- [ ] Restore purchases → tier reflects restored entitlements
- [ ] Cancel subscription → after expiry, tier reverts to `free`
- [ ] Web subscriber (backend tier) + no in-app purchase → still shows `premium` (merge logic)
- [ ] In-app subscriber + no backend tier → still shows `premium` (higher-of-two wins)

### 2.3 Subscription Source Display
- [ ] `subscriptionSource: 'app'` → Settings shows "via App Store / Google Play"
- [ ] `subscriptionSource: 'web'` → Settings shows "via e-rocks.com"

## 14. Performance & Stability

- [ ] Cold start to Doomscroll feed < 3s on mid-range device
- [ ] Scrolling any FlatList maintains 60 fps (no dropped frames)
- [ ] Navigating between 10+ screens in a single session — no memory leak / crash

## 15. Edge Cases

- [ ] Item with no images → placeholder shown, no crash
- [ ] Item with no price → "Price on request" or similar, no NaN
- [ ] Mineral with no specimens → empty gallery state
- [ ] Seller with 0 items → empty state in SellerItems
- [ ] Very long mineral/locality names → text truncates, doesn't overflow
- [ ] HTML entities in item titles → decoded correctly (he library)
- [ ] Negative or zero auction bid → handled gracefully
- [ ] API returns unexpected shape → error boundary catches, retry available
